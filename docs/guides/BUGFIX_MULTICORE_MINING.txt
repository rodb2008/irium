# MULTICORE MINING BUG FIX
**Date:** October 24, 2025
**Version:** Rust miner (current)
**Severity:** High - Wasted mining effort

## Summary
Legacy Python mining scripts are retired. The Rust miner now refreshes its block template whenever the node tip changes and restarts mining immediately, so other threads do not waste time on stale work.

## Root Causes (Legacy)
1. Slow node rescan interval
2. No coordination between miner processes
3. Height checked only once per block attempt

## Fixes Applied (Rust)
- Miner re-fetches `/rpc/getblocktemplate` when the node advances and logs template updates.
- Miner restarts nonce search when a new tip is detected.
- `--threads <n>` controls parallel hashing within a single process.

## Expected Results
- Stale work is discarded as soon as a new block arrives.
- Multiple cores mine the same current template, not outdated tips.

## Testing Instructions
1. Start the node: `./target/release/iriumd`
2. Start the miner: `./target/release/irium-miner --threads 4 --verbose`
3. Watch logs for template refreshes and new tip heights.

## Status
✅ Fixed and ready for testing

## Next Steps
- Verify with multiple nodes to confirm rapid template updates.
- Keep `IRIUM_MINER_STRICT_RPC=1` if you want mining to pause when RPC fails.
